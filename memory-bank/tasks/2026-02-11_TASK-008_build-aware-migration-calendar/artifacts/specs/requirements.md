# Requirements (Task-008)

## Source
Пользовательский блок требований в чате про:
- миграции при рассинхроне локальных данных и схем,
- build key/hash как триггер миграций,
- объединение данных из разных источников,
- динамику графиков во времени и календарную навигацию,
- динамические языки в редакторе опросников,
- подгрузку существующих переводов в translation panel.

## Acceptance
1. При смене build id выполняется reconciliation локальных данных без потери прогресса.
2. Импорт из разных источников не плодит дубли при совпадающем содержимом результата.
3. Графики показывают динамику по времени и поддерживают календарное представление.
4. В аналитике нет low/medium/high категорий; используется полная шкала USER-REQ.
5. В редакторе опросников языки вопросов управляются динамически.
6. Translation panel подгружает текущие переводы выбранного языка для редактирования.
