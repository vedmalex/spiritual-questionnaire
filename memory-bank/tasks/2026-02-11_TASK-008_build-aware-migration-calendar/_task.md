# TASK-008 — Build-Aware Migration and Calendar Dynamics

## Goal
Закрыть блок требований по build-aware миграциям, объединению данных и аналитике динамики во времени, а также добить динамические языки в редакторе опросников и подгрузку существующих переводов.

## Linked Requirements
- `UR-037`: Панель переводов загружает существующие переводы и поддерживает edit/add.
- `UR-039`: В редакторе опросников переключение/добавление языков создает и отображает соответствующие поля вопросов.
- `UR-042`: В аналитике используется полная шкала USER-REQ без категорий low/medium/high.
- `UR-047`: Миграция запускается при несовпадении локальной версии и версии приложения.
- `UR-048`: Используется build id/hash для определения необходимости миграции.
- `UR-049`: Объединение данных из разных источников выполняется через reconciliation с дедупликацией и конфликт-резолвом.
- `UR-050`: Графики показывают динамику оценок во времени.
- `UR-051`: Для навигации по динамике используется календарное представление.

## Deliverables
- Build-aware reconciliation в миграциях + сохранение build key в app state.
- Дедупликация импортируемых результатов по fingerprint при merge из разных источников.
- Новый `ScoreChart` без low/medium/high категорий, с полной шкалой и календарной динамикой.
- Динамические языки в `QuestionnaireEditor`.
- Автоподгрузка существующих переводов в `TranslationManager`.
